# Witty-Skill-Insight：从"感觉"到"量化"的进化之路

## 1. 背景：Agent使用Skill的五大困境

随着 Agent 技术的演进，我们正通过 **Skills** 赋予 LLM 实际行动的能力。Skill 已经成为 Agent 解决复杂问题的新时代大脑，其质量直接决定了 Agent 的上限。

然而，**Skill 从编写到运行的全生命周期中，开发者面临着五大核心困境**：

1. **观测维度单一，难以全面了解 Skill 的作用**：业界同类产品主要聚焦于 Agent 单次执行细节的展示，缺乏多维度的观测与对比——不同 Skill、不同框架、不同模型、不同用户任务之间的横向对比，以及效果、效率、成本、Skill 有效性等指标的综合分析。
2. **评估依赖人工经验，难以评估 Skill 有效性**：Agent 效果变好还是变差的根本原因依赖人工经验分析，标准不统一、效率低。缺乏一种更智能的评估机制，准确分析 Agent 行为变化与 Skill 之间的关联，为优化 Skill 提供依据。
3. **数据采集困难，侵入性强**：业界同类产品将 Agent 接入观测平台往往需要复杂配置或侵入式修改，数据采集门槛较高。
4. **高质量 Skill 编写难**：运维领域复杂任务存在多个步骤、指令与脚本，人工编写符合领域规范的高质量 Skill 效率低。
5. **大量 Skill 召回准确度低**：运维领域细分场景庞杂，大量 Skill 的描述信息相似时，精准召回困难。

**没有量化，就没有优化。** 这就是 Witty-Skill-Insight 存在的根本意义。它不仅要打开这个黑盒，更要**直观展示 Skill 优化前后，端到端的实际效果对比**。

## 2. 核心价值：从"体感评估"到"精准量化"

为了应对上述挑战，我们需要一套严谨的评估体系来透视 Agent 的行为。这套体系建立在**三个黄金指标**之上：

1. **效果 (Effectiveness) - 准确率**：
   Agent 是否真的解决了问题？结果是否可信？（不仅仅是看它说了什么，更要看它做了什么）。这是 Agent 存在的基石。
2. **效率 (Efficiency) - 时延**：
   解决问题的速度是否可接受？在生产运维场景中，响应时间直接决定了故障恢复的效率和用户体验。
3. **成本 (Cost) - Token**：
   达成目标消耗了多少算力？Token 使用量是否在经济可控范围内？对上下文窗口的占用是否合理？

![测试驱动开发](images/test_driven_develop.png)

通过对这三大指标的持续监控与分析，我们可以让每一次执行都被直观呈现，实现评估驱动开发，让 Agent 的进化有据可依。

## 3. 五大核心能力与技术方案

为了将上述理念落地，我们构建了 **Witty-Skill-Insight**，围绕五大核心能力逐一解决关键挑战：

### 核心能力一：Skill 自动生成

**对应挑战**：高质量 Skill 编写难

**问题**：运维领域复杂任务存在多个步骤、指令与脚本，人工编写符合领域规范的高质量 Skill 效率低。

**方案**：基于（一个或多个）案例文档自动提取 Skill 内容，对生成的多个 Skill 基于**文本聚类算法**进行相似度分析，提示用户并**自动合并**相似度高的 Skill，大幅降低 Skill 编写门槛。

### 核心能力二：Skill 自动评估与优化

**对应挑战**：评估依赖人工经验 + Skill 召回准确度低

**问题**：Agent 效果变化的根因依赖人工经验分析，标准不统一、效率低。大量 Skill 描述信息相似导致精准召回困难。

**方案**：基于**动静态评估与反思**（动态运行轨迹 + Skill 质量标准），对 Skill 进行全方位评估：

1. **智能裁判**：系统自动对比标准答案，对每一次执行进行打分，摆脱人工主观判断。
2. **Skill 深度归因**：引入智能归因系统，逐项分析评分损失，智能区分扣分项是**模型问题**（Skill 讲清楚了但模型没做）还是 **Skill 缺陷**（Skill 没说清楚或本身有误），并提供具体的改进建议。
3. **自动优化**：基于评估结果，自优化为符合领域规范的高质量 Skill，实现 Skill 迭代的闭环驱动。

### 核心能力三：透明代理，无感采集

**对应挑战**：数据采集困难，侵入性强

**问题**：业界同类产品将 Agent 接入观测平台往往需要复杂配置或侵入式修改，数据采集门槛较高。

**方案**：**不用写一行代码**即可开始采集运行期数据。实现无感、无侵入式的透明代理层，当前已支持主流 Agent 框架：

- **OpenCode** — 通过原生 Plugin 系统，毫秒级实时上报
- **OpenHands** — 透明代理模式，配置 Base URL 即可接入
- **Claude Code** — 通过底层日志监听与 Alias Hook 实现无感实时采集与同步

任何支持 OpenAI SDK 的 Agent 只需简单配置 Base URL 即可接入。它像是一个"隐形观察者"，在不打扰 Agent 运行的同时，精准捕获每一次对话、思考流（Chain of Thought）和工具调用。

### 核心能力四：多维对比

**对应挑战**：观测维度单一

**问题**：业界同类产品主要聚焦于 Agent 一次执行细节的展示，缺乏多维度的观测与对比能力。

**方案**：支持从 **Skill、框架、模型、用户任务**等维度，对准确率、Tokens、时延等关键指标进行横向对比与趋势分析。配合详细的工具调用序列回放，帮助开发者快速定位性能瓶颈。

### 核心能力五：深度分析

**对应挑战**：评估依赖人工经验

**问题**：Agent 效果变好/变差的根本原因依赖于人工经验分析，缺乏智能评估 Agent 行为变化与 Skill 之间关联的手段。

**方案**：基于对 Agent 执行过程的深度分析，**挖掘 Agent 行为变化与 Skill 之间的关联**，为后续优化提供精准依据。引入智能归因系统，逐项分析评分损失，智能区分扣分项是模型问题还是 Skill 缺陷，实现 Skill 优化的闭环。

---

*只有当能力被量化，优化才变得有迹可循*

## 4. 统一 Skill 生命周期管理

除了上述五大核心能力，Witty-Skill-Insight 还提供统一的 **Skill 生命周期管理**：

- **集中式 Skill 仓库**：跨框架统一存储与检索（OpenCode / Claude / OpenHands）
- **版本控制**：支持多版本管理（V1, V2...）与一键回滚
- **启用/禁用**：一键控制 Skill 生效状态
- **跨框架分发**：自动同步到不同 Agent 框架的本地目录

开发者可以轻松回滚到旧版本，或对比不同版本 Skill 的表现差异，让 Skill 迭代井井有条。
