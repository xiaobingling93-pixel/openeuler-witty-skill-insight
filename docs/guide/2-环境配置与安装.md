## 2. 安装平台

提供两种安装方式，按需选择。

### 2.1 一键安装（推荐）

适合快速体验，执行以下命令即可完成 Skill、观测平台、数据上报组件的安装：

```bash
npx @witty-ai/skill-insight install
```

按照提示完成安装后，即可在 `http://localhost:3000` 访问看板。

注意：若需要手动进行服务状态管理，参考如下启动和管理平台服务的常用命令：

```bash
# 启动服务（默认 3000 端口）
npx @witty-ai/skill-insight start

# 指定端口启动
npx @witty-ai/skill-insight start --port 3001

# 停止服务
npx @witty-ai/skill-insight stop --port 3000

# 重启服务
npx @witty-ai/skill-insight restart

# 查看服务状态
npx @witty-ai/skill-insight status

# 查看服务日志
npx @witty-ai/skill-insight logs
```

> 完整命令参考见 [附录 - 命令参考](7-附录-命令参考.md)。

### 2.2 源码安装

适合需要进行代码开发的场景。

```bash
# 克隆代码
git clone https://gitcode.com/openeuler/witty-skill-insight.git
cd witty-skill-insight

# 安装依赖
npm install

# 开发模式启动（自动初始化环境）
bash scripts/restart_dev.sh

# 生产模式启动（自动初始化环境）
bash scripts/restart.sh
```

## 3. 配置数据采集通道（可选）

如果 Agent 和平台部署在不同机器上，需要配置跨服务器的数据上报通道。在 Agent 所在机器的终端执行：

**Linux / macOS**

```bash
curl -sSf http://<IP>:<PORT>/api/setup | bash
```

**Windows**

```bash
irm http://<IP>:<PORT>/api/setup | iex
```

其中 `<IP>:<PORT>` 替换为平台部署服务的实际地址和端口。

### 配置步骤

**第一步：获取 API Key**

1. 打开看板页面（如 `http://192.168.1.100:3000`）
2. 点击右上角用户图标
3. 在弹出的对话框中获取或生成 API Key

**第二步：执行安装命令**

```bash
curl -sSf http://192.168.1.100:3000/api/setup | bash
```

安装过程会提示输入 API Key：

```
🔑 WITTY_INSIGHT_API_KEY is not set.
👉 Please enter your API Key: sk-xxxx-xxxx-xxxx
```

完成后，系统会自动写入配置文件 `~/.skill-insight/.env`，并自动安装相关数据上报组件。

## 4. 管理数据上报（可选）

安装完成后，如果因服务器重启等原因导致数据上报中断，可以手动管理上报通道：

```bash
# 启动所有数据上报通道
bash ~/.skill-insight/start_watchers.sh

# 关闭所有数据上报通道
bash ~/.skill-insight/stop_watchers.sh

# 单独启动 Claude Code 上报通道
bash ~/.skill-insight/start_claude_watcher.sh

# 单独启动 OpenClaw 上报通道
bash ~/.skill-insight/start_openclaw_watcher.sh

# 关闭 OpenCode 上报通道
rm ~/.opencode/plugins/Skill-Insight.ts

# 查看上报日志
tail -f ~/.skill-insight/logs/claude_watcher.log
tail -f ~/.skill-insight/logs/openclaw_watcher.log
```

## 5. 配置评估模型

评估模型用于对 Agent 执行结果进行自动评分。需要配置一个 LLM 作为评估模型。

### 5.1 添加配置

1. 登录看板，点击右上角 **⚙️ Eval Config** 按钮
2. 在弹出的对话框中，点击 **+ Add New Configuration**
3. 填写以下信息：

| 字段 | 说明 | 示例 |
|:---|:---|:---|
| Config Name | 配置名称 | 我的 DeepSeek |
| Provider | 模型提供商 | DeepSeek / OpenAI / Anthropic / Custom |
| Base URL | API 基础地址（可选） | `https://api.deepseek.com` |
| API Key | API 密钥 | sk-xxxx-xxxx-xxxx |
| Model Name | 模型名称 | deepseek-chat / gpt-4o |

4. 点击 **Test Connection & Save**，系统会自动测试连接并保存配置
5. 回到配置列表，点击 **Activate** 激活该配置

### 5.2 支持的模型提供商

| 提供商 | Base URL | 推荐模型 |
|:---|:---|:---|
| DeepSeek (Official) | `https://api.deepseek.com` | deepseek-chat |
| SiliconFlow | `https://api.siliconflow.cn/v1` | deepseek-ai/DeepSeek-V3 |
| OpenAI | `https://api.openai.com/v1` | gpt-4o |
| Anthropic | `https://api.anthropic.com/v1` | claude-3-5-sonnet-20240620 |
| Custom | 自定义 | 自定义 |

如使用其他兼容 OpenAI API 的模型（如火山引擎、智谱等），选择 **Custom** 作为 Provider，填写对应的 Base URL（不含 `/chat/completions` 后缀）。

> 建议使用能力较强的模型作为评估模型，以获得更准确的评测结果。

## 6. 数据库配置

### 6.1 默认配置（SQLite）

默认使用 SQLite，无需额外配置。数据库文件位于 `<项目根目录>/data/skill_insight.db`。

### 6.2 使用 OpenGauss

如需切换到 OpenGauss，编辑项目根目录下的 `.env` 文件：

```bash
# 注释掉 SQLite 配置
# DATABASE_URL="file:../data/skill_insight.db"

# 添加 OpenGauss 配置
DB_HOST=192.168.1.100
DB_PORT=5432
DB_NAME=skill_insight
DB_USER=witty
DB_PASSWORD=your_password
```

配置完成后，初始化数据库：

```bash
# 源码安装方式
python3 scripts/init_opengauss.py

# 或使用 Prisma 命令
npx prisma db push
npx prisma generate
```

### 环境变量说明

| 变量名 | 适用场景 | 说明 |
|:---|:---|:---|
| `DATABASE_URL` | SQLite | 数据库路径，如 `file:../data/skill_insight.db` |
| `DB_HOST` | OpenGauss | 数据库地址 |
| `DB_PORT` | OpenGauss | 数据库端口 |
| `DB_NAME` | OpenGauss | 数据库名称 |
| `DB_USER` | OpenGauss | 数据库用户名 |
| `DB_PASSWORD` | OpenGauss | 数据库密码 |
