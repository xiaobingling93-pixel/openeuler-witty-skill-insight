## `references/` — 技术原理与知识库

**核心用途：** 存储技术原理文档和知识库内容，用于增强 SKILL.md 描述。系统会读取此目录下的 Markdown 文件，为技能描述提供完整的文档结构参考。

**⚙️ 规范要求**

* **格式标准：** 所有文件使用 Markdown 格式（.md 扩展名）
* **结构化组织：** 建议按分类组织文档（如 API 文档、教程、最佳实践等）
* **索引文件：** 建议创建 `index.md` 作为目录索引，列出所有参考文档
* **内容完整性：** 文档应包含完整的技术说明、代码示例和最佳实践

**📄 目录结构示例**

```
references/
├── index.md              # 文档索引（推荐）
├── api.md                # API 参考文档
├── tutorial.md           # 教程文档
├── best_practices.md     # 最佳实践
└── examples.md           # 代码示例集合
```

**📝 使用说明**

* 系统会自动读取 `references/` 目录下的所有 `.md` 文件
* 这些文档会被用于增强 `SKILL.md` 的描述内容
* 文档内容应清晰、完整，便于 Agent 理解和引用
* 建议在 `SKILL.md` 中引用 `references/index.md` 作为完整文档结构的说明
